•	Маркетплейс объявлений:
•	Форма создания объявления: добавить поля: «Тип объявления» (выпадающий список: Продам/Куплю/Услуга/Другое) – это сохранится в базе и отобразится в списке; «Местоположение» (текст или выбор области); «Актуально до» (дата, по умолчанию +30 дней). Добавить клиентскую и серверную валидацию новых полей (например, дата не в прошлом).
•	Список объявлений: добавить фильтры: по типу (категории) – чекбоксы или выпадающий; по статусу (только активные – по умолчанию; возможно, вкладка «архивные» для завершённых); по ключевому слову – строка поиска, которая фильтрует по заголовку/описанию; по региону – если указано. Реализовать фильтрацию на клиенте (при небольшом количестве) или на сервере (лучше, если записей много).
•	Отображение объявлений: на карточке объявления (в списке) возле названия можно ставить метку типа (например, [Продам]), выводить город, дату размещения. Новый пост – помечать как «NEW» первые пару дней визуально.
•	Страница объявления: переделать отображение предложений поставщиков:
o	Если текущий пользователь = автор объявления, показывать список всех предложений: имя поставщика, его комментарий/цену, кнопки «Принять» и «Отклонить» для каждого. При нажатии «Принять» – устанавливать у объявления статус «closed/выполнено», сохранять id выбранного предложения (победителя), отправлять уведомления победителю и остальным.
o	Если пользователь = поставщик, не автор, показывать только своё предложение (если он уже отправил). Чужие предложения ему не показывать. Если он ещё не отправил – форму ввода предложения (уже есть). После отправки формы либо сразу отображать своё предложение, либо серым текстом «Ваше предложение отправлено, ожидайте решения».
o	Если пользователь = админ, он может видеть все предложения (как автор) и имеет право закрыть объявление или удалить.
o	Обеспечить, чтобы при закрытии объявления (либо вручную автором, либо по принятию предложения) на странице объявления отображался статус «Закрыто» и новые заявки уже нельзя было добавить (дизэйблить форму).
o	Если объявление закрыто, но победитель выбран, можно отображать: «Исполнитель: Компания ХХХ» (если не противоречит приватности).
o	Добавить возможность автору закрыть объявление без выбора (например, если передумал) – кнопка «Закрыть объявление» с подтверждением. Она просто ставит статус «закрыто», уведомляет поставщиков «Объявление закрыто без выбора исполнителя».
•	Профиль пользователя (кабинет):
o	В разделе профиля добавить страницу «Мои объявления»: список объявлений пользователя (заголовок, статус, ссылка на редактирование/просмотр). Там же кнопка «Создать новое объявление».
o	Добавить страницу «Мои отклики» для поставщика: список объявлений, на которые он подал предложение, с статусом (ожидает / проигран / выигран). Статус можно определять: если объявление закрыто и выбрано не его предложение – «не выбрано», если выбрано его – «Ваше предложение принято!».
o	В профиле отображать контактные данные пользователя, дать возможность их отредактировать (минимум – имя, организация, email, телефон). Возможно, эти данные будут видны автору объявления при выборе его предложения, так что надо их собирать. Если сейчас при регистрации не спрашивается организация/телефон, можно добавить поля в профиль.
•	Админ-панель для объявлений:
o	Добавить раздел «Объявления» со списком всех объявлений, возможностью фильтровать по статусу/типу. Админ может удалять или скрывать объявления, редактировать текст (если, например, нужно убрать контакты из тела – чтобы все контакты шли через систему).
o	Добавить управление пользователями: возможность выставить роль (например, переключать «поставщик/заказчик» если нужно, хотя можно обойтись без), бан пользователя (чтобы он не мог логиниться, если нарушил правила).
o	Модерация: при создании нового объявления флаг approved = false (не опубликовано) – админ в панели видит «Новые объявления (3)», заходит, проверяет содержимое, нажимает «Одобрить». После этого approved=true, оно становится видимо всем. Если такой функционал согласован – внедрить. Если решили без премодерации, всё равно предусмотреть, что админ может снять с публикации (toggle видимости).
•	Уведомления:
o	Настроить NotificationSystem так, чтобы при ключевых событиях создавались уведомления:
o	Новая заявка на ваше объявление (для автора).
o	Ваше предложение принято/отклонено (для поставщика).
o	Ваше объявление одобрено админом или удалено.
o	Реализовать эти уведомления на бекенде (сохранение в таблицу notifications с ссылкой на пользователя).
o	На фронте – иконка колокольчика, при клике – выпадающий список уведомлений. Уведомление содержит короткий текст и ссылку (например, «Новое предложение к вашему объявлению "Купить огнетушители"» – клик ведёт на AdPage данного объявления).
o	Если возможно, подключить WebSocket для мгновенного оповещения. Если сложно – реализовать polling: периодически запрашивать новые уведомления. Библиотека EnhancedNotifications возможно уже готова к этому, нужно задействовать.
