Задача: довести сайт NewsFire до «производственного» уровня по калькуляторам, печати отчётов, SEO/безопасности и админ-UX.

Обязательные требования (приёмка):

Подключить client/src/components/ui/print.css глобально так, чтобы стили печати применялись ко всем калькуляторам.

Во всех калькуляторах (FireExtinguisher, NGPS, FireLoad, Evacuation, ExplosionCategory, FireRisk) добавить унифицированный Report-блок внизу:

«Вводные» (все входные параметры),

«Формулы» (чётко, с подстановкой чисел),

«Промежуточные расчёты»,

«Итог» (акцент),

«Обоснование» с пронумерованными ссылками на CalculatorsMethodology (якоря по разделам),

QR/ссылка на расчёт. Кнопка Печать (PrintButton) печатает только этот блок в A4, с корректными переносами.

Эвакуация:

Добавить справочник максимально допустимых расстояний до выхода по типам зданий. Если введённое L превышает — выводить красный алерт.

Реализовать режим нескольких выходов (2–3): распределение N, суммарная пропускная способность; итог = макс по выходам/пути.

Если doorWidth < 0.8 — явный алерт «минимум 0,8 м не соблюдён».

Категории А/Б/В/Г/Д:

Внедрить расчётный критерий давления (или эквивалент) и правило: если предсказанное избыточное давление ≥5 кПа — категория А.

Поддержать Г (малые количества в нагретом состоянии) и Д (негорючие): при нулевой пожарной нагрузке/отсутствии горючих выдавать Д.

Кнопка «Очистить всё» для списка веществ.

NGPS: расширить MANDATORY_INDUSTRIES по Пост. №1017 (проверить полный перечень); добавить подсказки к полям («площадь суммарно по А/Б/В», «расстояние по прямой»).

SEO: вставить SEOHead на страницы PostPage, AdPage, MarketplacePage, SectionPage, SubsectionPage (динамический title/description из данных). Добавить robots.txt и sitemap.xml (генерация из БД).

Безопасность: обернуть вывод HTML на PostPage.tsx через DOMPurify (серверный/клиентский санитайз).

Методичка: проставить якоря по калькуляторам и кнопку «Скачать PDF» (можно статичный PDF). Указать версию и дату актуальности.

Calculators list: добавить карточку «Пожарная нагрузка» на /calculators.

Админ-UX: сделать боковое меню (Секции/Подсекции/Посты/Документы/Объявления/Заявки/Пользователи/Модерация/Медиа/Уведомления/Аналитика), в таблицах — фильтры/поиск/сортировки, у сущностей — бейджи статусов.

Нефункциональные:

Поддержать мобильную печать (поля, отступы A4).

Unit-тесты на чистые функции калькуляторов (пара эталонных кейсов на каждую формулу).

Готовность: PR с чек-листом выполнения пунктов 1–10 и скриншотами «до/после»; демо-видео печати «паспортов расчёта».
